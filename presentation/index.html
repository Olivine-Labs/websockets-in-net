Why WebSockets?

Other Technologies

  • Ajax - inefficient, needless server communication
  • Comet - inefficient, hack for push notifications
  • Flash - extra toolset, additional runtime

What these have in common - emulating a multiplex socket.

Use Cases

  • Games - real-time game state
  • Collaboration - real-time application data for collaboration
  • Push Notifications - system alerts, chat from users

All of these things were possible before WebSockets, with much more complexity.
